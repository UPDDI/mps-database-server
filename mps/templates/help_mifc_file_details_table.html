  • Details on the MIF-C File Format

    The following table provided details on the MIF-C (-a and -i) file formats.

    Column Header Applies to File Column Heading Required Content Required Description Example (Assays) Example (Images)
    Chip ID or Well ID MIF-C-a and MIF-C-i yes required Primarily identifier used to refer to a specific instance of an MPS model. In cases where wells are being used as a static comparison to a chip, for purposes of entry into the MPS-Db, each of those wells will need to be assigned a Chip ID, which could be “PlateID-WellID” (e.g. P1234-B05). c00001 c00001
    Assay Plate ID MIF-C-a and MIF-C-i yes no A unique ID assigned to a well plate (the enter plate). Recommended naming convention is as follows: “P” & “yyyy” & “-“ & “dddd” & ”a string of the user’s choice” (e.g., P2017-0223A). PT_HIM31_02022017 PT_HIM31_02022017
    Assay Well ID MIF-C-a and MIF-C-i yes no When used in the context of an Assay (Assay Well ID), specifies the location in a well plate where media and/or standard was placed and observed via a plate reader (e.g., A01, A15, B13). A04 none
    Assay Method MIF-C-a and MIF-C-i yes yes Indicates the method (e.g., SW-846 for a group of metals) or kit (e.g., Ammoniagenesis (Abcam: ab83360) used to measure a Target/Analyte. Must be in MPS-Db list. In the case of a kit, the kit, and, if the kit can be used with different options, the option used should be specified. Ammonia Assay Kit (abcam: ab83360) LIVE/DEAD Viability/Cytotoxicity Kit for mammalian cells (Molecular Probes: 03224 - Calcein AM/EthD-1), Hoechst, Phase Contrast
    Assay Target MIF-C-a and MIF-C-i yes yes The focus of an assay, as in, what is being measured. Examples include: Albumin, Lactate Dehydrogenase, TNF-Alpha. Must be in MPS-Db list. Ammonium Nucleus | Live Cells | Dead Cells | Tissue Structure
    Subtarget MIF-C-a and MIF-C-i no no When needed (if data are more appropriately displayed on a heatmap), else will be populated with 'none' in the MPS. none (in TempO-Seq, would be the gene) none
    Sample Location MIF-C-a and MIF-C-i yes yes Specifies a compartment in the MPS model such as: brain, vascular, neural, apical, basal. Must be in MPS-Db llist. When a sample is collected from the MPS model, the compartment should be combined with a prefix indicating where the sample was collected, such as “effluent-brain” or “stock-brain”. Note that, when a chamber is image, no prefix would be used. effluent-apical Apical
    Day MIF-C-a and MIF-C-i yes no The time, from the start of the experiment, when a Sample was collected and/or when a Measurement was made (e.g, an image taken). The sample time is imported into the MPS-Db in Day, Hour, and Minute. Nulls (blanks) will default to 0 on upload. You may use decimal places..for example, D 0 H 48.5 M 0 = D 2 H 0.5 M 0 = D 2 H 0 M 30 24 24
    Hour MIF-C-a and MIF-C-i yes no See Description for Day (above). 2 2
    Minute MIF-C-a and MIF-C-i yes yes See Description for Day (above). 0 0
    Value MIF-C-a and MIF-C-i yes no In the context of Processed Data, indicates the amount observed. 0.13036162084885994 na
    Assay Unit MIF-C-a and MIF-C-i yes yes In the context of the Processed Data, indicates the unit of the amount observed. mmol/L image' for images, 'video' for videos
    Caution Flag MIF-C-a no no A flag indicating the status of a reported result. See the online glossary for list of valid caution flags. W na
    Exclude MIF-C-a no no When the Exclude field of the MPS-Db is populated (X), that data point will not shown, by default, in graphs or be provided in exports from the MPS-Db. X na
    Duration Sample Collection (days) MIF-C-a no no The amount of time sample effluent was collected (days). 1 na
    Duration Sample Collection (hours) MIF-C-a no no The amount of time sample effluent was collected (hours). 0 na
    Duration Sample Collection (minutes) MIF-C-a no no The amount of time sample effluent was collected (minutes). 0 na
    Sample Volume MIF-C-a no no The volume of sample collected. 240 na
    Sample Volume Unit MIF-C-a no no The unit of the volume of the sample collected. µL na
    Help Notes: MIF-C-a and MIF-C-i no no, but highly recommended, especially for images In the context of Processed Data, indicates something of interest with regard to the value being reported. For images, indicates what the colors in the images indicate. Well was empty. The image shows cell nucleus. ChipA is on the left and ChipB is on the right.
    Replicate MIF-C-a and MIF-C-i no no At the assay level, multiple measurements of the same endpoint (i.e., LDH) observed on the same media sampled and observed at the same time. 1 1
    Cross Reference MIF-C-a and MIF-C-i no no In the context of the processed data (the MIF-C file), the cross reference is a string of text that is used to cross reference data points in the MPS-Db to other database systems (i.e. LabKey). 12345678-03 nan
    Results Group MIF-C-a and MIF-C-i no no This is an optional link to results information. 105 nan
    Image File Name MIF-C-i yes yes The file name of the image that will be provided to the MPS-Db Center via a drop box or exteral storage device. The naming of the file is up to the data provider, but a descriptive naming protocol is highly recommended. na Tamu_PT_UW_d24h0m0_C00039_S01_W5.tif
    Image Field MIF-C-i yes yes An indication of where in the chamber of the device the image was taken. Use integers and increment for each additional location. na 1
    Image Field Description MIF-C-i no no Provide a description of what is captured in the image field. If multiple images were taken at various locations, indicate something about where/how the location was selected (e.g., top left corner, center of channel, etc…) na Image field manually selected.
    Image Magnification MIF-C-i yes yes The image magnification as an integer. na 20
    Image Resolution MIF-C-i yes yes The resolution of one pixel of the image. na 0.077
    Image Resolution Unit MIF-C-i yes yes The unit of the resolution. na µm
    Image Sample Label MIF-C-i yes yes The dye/stain that will cause the fluorescence observed in the image. Must exist in MPS-Db Sample Label list. na Hoechst | Calcein AM | EthD-1 | Unlabeled
    Image Sample Label Description MIF-C-i no no Provide a description of the image source used. na 0
    Image Wavelength (ex/em nm) MIF-C-i no no Wavelength of the illuminating light. na 405/- | 488/- | 633/- | White Light
    Image Color Mapping MIF-C-i yes yes The color displayed in the image. na blue | green | red | phase contrast
    Image Setting Note MIF-C-i no no An optional note regarding the image settings. na W5-FITC, DAPI, TRITC, White Light
