Global Database Tools

  • Global Database Search

    • Use a Global Database Search tool to search the whole MPS-Db from the Home page. To search:
      • Go to the MPS-Db Home page
      • Type a search term into the global search bar
      • Type a space and add an additional term(s) (optional)
      • Make preliminary selection (hit enter, click the blue Search button, or mouse down the options list to item of interest and click the left mouse button)
      • Wait for a list of matching information to appear
      • Click to check (or uncheck) filter boxes in the filter section to further restrict (or unrestrict) matching results (optional)
      • Click on provided link to be routed to desired location

  • Table Searching and Exporting

    • In many places throughout the database, there are lists/tables with a search bar located at the top. To search:
      • Type a search term in the search bar
      • Type a space and add an additional term(s) (optional)
      • From the list, click on the View/Edit button or click an active link to be routed to desired location

    • Help Notes:
      • Above search bar are options to copy the table to the clipboard, download the table to CSV, or print the table
      • The Column visibility button can be used to turn columns on/off in the list/table
      • To sort the list/table, click on the arrow in the column header
      • To sort on multiple columns, click on the arrow on the primary sort column, then hold the shift key and click on additional column arrows
      • Click the blue to see hidden fields

  • Download Button

    • There are many locations in the database where a Download button is provided. To download:
      • Click the Download button to download data
      • If shown, check/uncheck box for changing data filtering options (optional)
      • Use options for filtering, if provided (optional)

    Study Design, Set-up and Test Compound Selection Using the MPS-Db (20201110)

    UPDDI's Dr. Lawrence Vernetti presents the webinar, Study Design, Set-up and Test Compound Selection Using the MPS-Db. This webinar demonstrates the process of designing an MPS-based toxicology Study using the MPS-Db. Dr. Vernetti demonstrates how to use multiple features to review preclinical, clinical and adverse event data that are available in the MPS-Db, and show how those data can be used to guide the selection of clinical drug control compounds to test in an MPS experimental model. Webinar was recorded on November 10, 2020 and is approximately 60 minutes long. Questions asked during the webinar, along with answers, are available below.

    • Questions and Answers

      Question Answer
      Is all the data in the MPS database for small molecules or even biologics/antibodies included? Presently, it is predominantly small molecules. We add compounds as they are used in designing studies and are very interested in adding biologics/antibodies as they are used in Study designs.
      Are recent clinical compounds or one's which were stopped in different phase trails also included in this database? It is easy to add any compound to the database, which automatically retrieves the compound properties Dr. Vernetti indicated. We don't currently add all compounds, rather we add them as needed. However, the compound list in the Adverse Events feature comes from the FDA FAERS database, so there are more compounds in that data, but it is post marketing. We are working on an interface to automatically pull in Clinical Trial data and when that is complete it will include an expanded list of recent and failed clinical compounds.

    • Assay Data Visualization

      {{ glossary_dict.assaydatavisualizationexcludingomic_def|linebreaks }}

      Assay Data Visualization (Intra-Study)

      The Assay Data Visualization (Intra-Study) feature is accessed from within an individual Study, pre-filtered as follows:
      1) each {{ glossary_dict.studytreatmentgroup_term }},
      2) each plate (if using a plate),
      3) each chip or well, and
      4) the whole Study.

      • Assay Data Visualization - Each Treatment Group:
        • Go to the Studies tab/icon
        • Click on the View/Edit button next to the Study of interest
        • Click on the View button next to the group of interest
        • Review metadata and, if displayed, the {{ glossary_dict.assaydatapoint_term }} graphs

      • Assay Data Visualization - Each Plate:
        • Go to the Studies tab/icon
        • Click on the View/Edit button next to the Study of interest
        • Scroll down to the Plates section and click on the View/Edit button next to the plate of interest
        • Review metadata and, if displayed, the {{ glossary_dict.assaydatapoint_term }} graphs

      • Assay Data Visualization - Each Chip or Well:
        • Go to the Studies tab/icon
        • Click on the View/Edit button next to the Study of interest
        • Scroll down to the Chips and Wells section and click on the View/Edit button next to the chip/well of interest
        • Review metadata and, if displayed, the {{ glossary_dict.assaydatapoint_term }} graphs
        • Review the {{ glossary_dict.assaydatapoint_term }} table
        • Note: The {{ glossary_dict.assaydatapoint_term }} table offers the {{ glossary_dict.datagroupeditor_term }} and {{ glossary_dict.datagroupadministrator_term }} of the Study the ability to include/Exclude an {{ glossary_dict.assaydatapoint_term }} from being included/Excluded from graphs and analyses by default


      • Assay Data Visualization - Whole Study:
        • Go to the Studies tab/icon
        • Click on the View/Edit button next to the Study of interest
        • Click the Analyze/Edit button and select the Study Summary option
        • Review metadata and, if displayed, the {{ glossary_dict.assaydatapoint_term }} graphs

      • There are many options available for customizing graphs, to learn more, click on a link below:
        • Custom Filtering for Data Visualization (sidebar)
        • Custom Graphing for Data Visualization (sidebar)
        • Customizing Individual Graphs

      Assay Data Visualization (Inter-Study)

      • To perform inter-study visualization:
        • Go to the Data Analysis tab/icon
        • Select the Graphing/Reproducibility option
        • Check at least one box in each list (work left to right, top to bottom)
          • Notice how checking and unchecking boxes changes the number of points ({{ glossary_dict.assaydatapoint_term }}s) that are selected for inclusion
        • Click on the Show Plots button
        • Scroll down the page to see plots data
        • Keep scrolling down the page to see the group metadata

      • There are many options available for customizing graphs, to learn more, click on a link below:
        • Custom Filtering for Data Visualization (sidebar)

    • Reproducibility Analysis

      {{ glossary_dict.reproducibilityanalysis_def|linebreaks }}

      Reproducibility Analysis (Intra-Study)

      • To perform intra-study reproducibility analysis:
        • Go to the Studies tab/icon
        • Click on the View/Edit button next to the Study of interest
        • Click the Analyze/Edit button and select the Reproducibility option
        • Use the Custom Filtering for Data Visualization (sidebar) to change grouping or filtering of data
        • Check the box next to the set of interest to see the results of interest
        • Scroll down page to detailed results section of interest (including chips, studies and each Study's reproducibility status, Max CV, ICC, and graphs)

      Reproducibility Analysis (Inter-Study)

      • To perform inter-study reproducibility analysis:
        • Go to the Data Analysis tab/icon
        • Select the Graphing/Reproducibility option
        • Check at least one box in each list (work left to right, top to bottom)
          • Notice how checking and unchecking boxes changes the number of {{ glossary_dict.assaydatapoint_term }}s selected for inclusion
        • Click on the Show Reproducibility button
        • Use the Custom Filtering for Data Visualization (sidebar) to change grouping or filtering of data
        • Check the box next to the set of interest to see the results of interest
        • Scroll down page to detailed results section of interest (including chips, studies and each Study's reproducibility status, Max CV, ICC, and graphs)
        • Explore the Reproducibility Properties options in the sidebar (optional)
        • Customized properties for {{ glossary_dict.reproducibilityanalysis_term }} are available in the MPS-Db via a sidebar on the inter-study {{ glossary_dict.reproducibilityanalysis_term }} page
        • To perform {{ glossary_dict.reproducibilityanalysis_term }}:
          • Reproducibility Comparison within Selected Data Sets - By Center (Center to Center) or By Study (Study to Study)
          • Normalize by Median - helpful if trends are similar but values are on different scales
          • Maximum Interpolation Size - maximum number of data points that can be interpolated, between experimental measurements, to allow for comparison within selected data sets, center to center or Study to Study (shown as stars on the reproducibility plots)

      • Help Notes:
        • By default, the screen will automatically refresh, to refresh manually, check the Manually Refresh box, make the desired changes, then click on the Refresh button to apply changes

    • Study Sets

      {{ glossary_dict.studyset_def|linebreaks }}
      • To add a {{ glossary_dict.studyset_term }}:
        • Go to the Studies tab/icon
        • Select the Add Study Set option
        • Select the studies to include in the set by putting a check mark in the box
        • Click the Continue button
        • Fill out the form
        • Click the Select Assays button to limit data to include in the {{ glossary_dict.studyset_term }} (optional)
        • Click the Submit button to save

      • Help Notes:
        • A {{ glossary_dict.studyset_term }} is conceptually similar to an inter-study analysis, with the added functionality to:
          • Save the selection of the {{ glossary_dict.studyset_term }} for later retrieval
          • Assign the {{ glossary_dict.studyset_term }} a URL for easy sharing (e.g., inclusion in a publication)
          • Data from each Study in the {{ glossary_dict.studyset_term }} can be limited to a subset of one or more {{ glossary_dict.assaytarget_term }}s
        • Viewers of a {{ glossary_dict.studyset_term }} created by another user will only see data from Studies to which they have access (i.e. a {{ glossary_dict.studyset_term }} does NOT override existing permissions to Studies)

    • Collaborators (Collaborator Groups)

      {{ glossary_dict.collaboratorgroup_def|linebreaks }}
      • About {{ glossary_dict.collaboratorgroup_term }}s:
        • A {{ glossary_dict.collaboratorgroup_term }} is a means of allowing data from a private Study (a Study that has not been signed off) to be shared with another user who is not a member of the same {{ glossary_dict.datagroup_term }} (e.g., someone from another lab)
        • Using a {{ glossary_dict.collaboratorgroup_term }} is currently the only way to allow someone from another data team to see data contained in a Study prior to either signing off a Study or opening the Study to the public
        • For more information on how to share data using {{ glossary_dict.collaboratorgroup_term }}s click here

    • Accessors (Access Groups)

      {{ glossary_dict.accessgroup_def|linebreaks }}
      • About {{ glossary_dict.accessgroup_term }}s:
        • An {{ glossary_dict.accessgroup_term }} is a means of sharing data from a Study during the year embargo period (after sign-off and before public release) with colleagues who are not members of the same {{ glossary_dict.datagroup_term }} (e.g., someone from another lab)
        • For more information on how to share data using {{ glossary_dict.collaboratorgroup_term }}s click here

      {{ glossary_dict.pbpkanalysis_def|linebreaks }}

        • For a brief introduction to using this feature, see Study Design, Set-up and Test Compound Selection Using the MPS-Db (time 45:03 minutes)

    Customization Options

    • Custom Grouping and Filtering (sidebar)

        Graph customization is not saved, it lasts only until the user navigates away from the page.

      • Customized grouping and filtering is available in the MPS-Db via a sidebar where data are available for review. To customize:
        • If hidden, click the Toggle Sidebar button to show the sidebar
        • Click on the parameter menus (Study Parameters, Cell Parameters, Compound Parameters, Setting Parameters) to see options
        • To filter - click on the filter symbol
          • To select data, check the box next to the data

    • Validation, Flags and Notes (optional)

      To facilitate data review and validation, the database offers the option to flag and add notes to a 1) Study, 2) {{ glossary_dict.studyitem_term }} (i.e. a chip or a well), and 3) plate. When a Study is flagged, a flag is display in the Review column for that Study in the Study list (the note is displayed as a hover tip). When a {{ glossary_dict.studyitem_term }} is flagged, the flag is displayed in the Review column for that {{ glossary_dict.studyitem_term }} in the {{ glossary_dict.studyitem_term }} list. In addition, a {{ glossary_dict.studyitem_term }} can be 'validated' (a check mark will show in the Review column for the {{ glossary_dict.studyitem_term }} in the {{ glossary_dict.studyitem_term }} list).
