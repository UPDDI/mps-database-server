    No Format Selected - Assumptions

  • Files are comma delimited (CSV), tab delimited (TSV), or space delimited text files.
  • CSV files do not contain commas in their data fields (use TSV).
  • There is only one plate size per file.
  • Data blocks are in the same dimensions as the plate size.
  • Data blocks are stacked vertically (above/below), not horizontal (side-by-side).
  • Data blocks are separated by a blank line (best results are obtained when the blank line is tagged ~blank in the first column.
  • A data block is considered to end one line above the tagged blank line (or the first blank line if there is more than one).

    • Customize by Setting Format Information

    If computer returned incorrect results on first run, turn ON the set to and enter the correct format information, the redetect the blocks. Currently only works for vertically stacked data blocks (not side-by0side).

  • Set the plate size for better performance.
  • Set the number columns to ignore (for best results, label the data block headers with 1, 2, 3, etc.)
  • Set the file delimiter.
  • As a last resort, set the number of data blocks (will likely require manual editing of the block information).
