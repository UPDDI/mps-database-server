    Tips and Underlying Assumptions
  • Files are comma separated/delimited (CSV), tab separated/delimited (TSV), or space separated/delimited text files.
  • CSV files do not contain commas in their data fields (if they do, use TSV).
  • There is one plate size per file (i.e. all data blocks are the same size).
  • Data blocks are in the same dimensions as the plate (e.g., a 96 well plate is 8 rows by 12 columns).
  • Data blocks are separated by a blank line (best results are obtained when the blank line is tagged ~blank or ~end in the first column of the file) or the plate size is specified and # Columns to Ignore is set to '123'.
  • If blank lines are used to locate data blocks, a data block is considered to end one line above the tagged blank line (or the first blank line, if there is more than one).
  • Unless plate size is set and # Columns to Ignore is set to '123', or a specific file format has been programmed to detect side-by-side data blocks, only the left most set of data blocks will be detected (additional blocks located to the right will be ignored).
  • Select the File Format most like the data file and click to detect file blocks. If incorrect results are returned on the first run, click to Show Settings, set some of the format information manually (and turn On the corresponding Set slider), then click to Redetect blocks.
  • Setting the number of data blocks well return the specified number of blocks. The block information can be edited manually after detection.
